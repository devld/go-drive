# 路径属性与挂载

## 路径属性

在“管理员 → 路径属性”可为任意虚拟路径配置：

| 属性 | 作用 |
| --- | --- |
| 密码 | 访问目录及其内容前要求输入路径密码 |
| 默认排序 | 进入目录时使用指定排序方式 |
| 默认模式 | 列表或缩略图模式 |
| 隐藏模式 | 使用路径模式隐藏匹配条目 |

每个属性都有独立的“递归”开关。开启后，属性应用于子路径；更具体路径的配置可以覆盖父路径。浏览器会把当前会话中已输入的路径密码暂存在 `sessionStorage`，关闭会话后需要重新输入。

隐藏模式只是界面/路径层的过滤手段，不应代替权限控制。敏感文件仍应配置拒绝读取权限。

路径模式语法见[路径模式参考](../reference/path-patterns.html)。

## 创建挂载

管理员在文件或目录的右键菜单中选择“挂载到”，再选择目标位置。挂载条目前会显示 `@` 标记（仅管理员可见）。

挂载不会复制数据；同一真实条目可以在虚拟目录树的多个位置出现。允许递归挂载，但应避免制造难以理解的目录结构。

## 权限和属性

- 挂载位置使用挂载位置的权限，而不是源路径权限。
- 路径属性同样根据用户正在访问的虚拟路径解析。
- 删除挂载只删除挂载关系，不删除源文件；删除挂载下的真实条目则会修改源数据。
- Drive 被删除或路径不存在后，可在维护页清理无效挂载和权限。
