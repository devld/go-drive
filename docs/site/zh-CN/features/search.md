# 搜索与索引

搜索默认关闭。启用当前 SQLite 搜索器：

```yaml
search:
  enabled: true
  type: sqlite
```

重启 go-drive 后，在“管理员 → 其他 → 文件索引”创建索引任务。旧版本使用的 `bleve` 已不再支持。

## 首次索引

路径留空会从虚拟根目录开始索引；也可以只索引某个子路径。任务在后台执行，页面显示状态和进度，并可中止未完成任务。

以下情况应重新索引相关路径：

- 首次启用搜索。
- 新增、删除或重命名 Drive。
- 文件由 go-drive 外部直接修改。
- 改变索引过滤规则。
- 恢复数据库或迁移存储。

通过 go-drive 完成的日常文件操作会更新索引，但外部系统的变化无法自动感知。

## 过滤规则

每行以 `+` 或 `-` 开头，后面使用[路径模式](../reference/path-patterns.html)。匹配不区分大小写。

```text
-**/.git/**
-**/node_modules/**
-**/*.tmp
+public/**
```

- `-`：排除匹配路径。
- `+`：包含匹配路径；只要存在任意 `+` 规则，不匹配任何 `+` 的路径都会被排除。
- 排除规则优先：路径同时匹配 `+` 和 `-` 时排除。

修改过滤规则后重新索引，旧索引不会仅凭配置变化自动清理。

## 搜索权限

搜索结果仍受用户根路径、路径权限和隐藏规则约束。管理员测试成功不代表匿名用户或普通用户能看到相同结果。
